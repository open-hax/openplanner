# Status Bar

**Parent**: `knowledge-ops-workbench-ux-breakdown.md`
**Epic**: 0. Shell Foundation
**Points**: 2
**Dependencies**: 0.1 Shell Layout Component, P1A (tenant enforcement for collection context)

---

## Purpose

Bottom status bar showing active collection, LLM provider, token budget, agent status, and keyboard mode.

---

## Scope

Create the status bar component that provides at-a-glance context about current system state.

---

## Files to Create

| File | Purpose |
|------|---------|
| `src/shell/StatusBar.tsx` | Status bar component |
| `src/shell/status-hooks.ts` | Hooks for subscribing to status state |

---

## Behavior

### Status Items (left to right)

| Item | Content | Source |
|------|---------|--------|
| Collection | Active collection name | P1A tenant context |
| Model | Current LLM provider/model | Proxx config |
| Tokens | Token budget remaining | Session state |
| Agent | Active agent run count | Agent runtime |
| Mode | Current keyboard mode | ChordOverlay state |

### Display Rules

- If no collection selected: show "No collection"
- If no model: show "No model"
- If no active agents: hide agent indicator
- Token budget: show as "X / Y tokens" or "X tokens remaining"
- Mode: show "Normal" or current chord prefix (e.g., "SPC...")

---

## Exit Criteria

- [ ] Shows active collection name (or "No collection")
- [ ] Shows current LLM provider/model
- [ ] Shows token budget (when available)
- [ ] Shows agent run count when agents active
- [ ] Shows current keyboard mode
- [ ] Responsive: items collapse gracefully on narrow viewports
- [ ] Dark mode supported

---

## Out of Scope

- Editing any status value
- Detailed agent run info (see Agent Workspace)
- Collection switcher (use Context Bar)

---

## Acceptance Tests

1. Load with no collection → shows "No collection"
2. Select collection → shows collection name
3. Start agent run → count appears
4. Complete agent run → count disappears
5. Press SPC → mode shows "SPC..."
6. Resize to 375px → status bar remains readable
