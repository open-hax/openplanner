Create the content editor page structure that hosts the document editing experience.

## Behavior

### Layout

```
┌──────────────────────────────────────────────────────────────────┐
│ ← Back    [document title]                    [Save] [Publish ▾] │
├──────────────────────────────────────┬───────────────────────────┤
│  Title                              │  INSPECTION PANEL          │
│  ┌──────────────────────────────────┐│                           │
│  │ [editable title]                 ││  (see 2.3 Provenance)     │
│  └──────────────────────────────────┘│                           │
│                                      │                           │
│  Body                                │                           │
│  ┌──────────────────────────────────┐│                           │
│  │                                  ││                           │
│  │  [markdown or rich text editor]  ││                           │
│  │                                  ││                           │
│  │                                  ││                           │
│  └──────────────────────────────────┘│                           │
│                                      │                           │
│  Structured fields                   │                           │
│  collection: [dropdown ▾]            │                           │
│  visibility: [dropdown ▾]            │                           │
│  status: [badge]                     │                           │
└──────────────────────────────────────┴───────────────────────────┘
```

### Structured Fields

| Field | Options | Required |
|-------|---------|----------|
| Collection | List from P1A context | Yes |
| Visibility | internal, review, public, archived | Yes (default: internal) |
| Status | draft, in_review, published | Auto from visibility |

### Editor Type

- Markdown editor (simpler to implement)
- Future: rich text (Out of scope for this spec)

---

## Exit Criteria

- [ ] Document title editable
- [ ] Body editor accepts markdown input
- [ ] Collection selector shows available collections
- [ ] Visibility selector shows options
- [ ] Status indicator shows current state
- [ ] Save button persists changes
- [ ] Publish menu shows workflow options
- [ ] URL param `:id` loads existing document
- [ ] No `:id` creates new document

---

## Out of Scope

- AI suggestions (see 2.2)
- Provenance panel content (see 2.3)
- Publish workflow (see 2.4)
- Rich text editing

---

## Acceptance Tests

1. Navigate to `/content/new` → empty editor with default values
2. Navigate to `/content/abc123` → loads document
3. Edit title → saves on blur or Save button
4. Select collection → persists selection
5. Change visibility → status badge updates
