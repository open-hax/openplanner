## Purpose

Show counts for review queue, approval queue, and policy violations with CTAs.

---

## Scope

Create the attention card components that surface "what needs my attention" on the dashboard.

---

## Files to Create

| File | Purpose |
|------|---------|
| `src/pages/DashboardPage.tsx` | Dashboard page component |
| `src/components/dashboard/AttentionCard.tsx` | Reusable attention card |

---

## Behavior

### Attention Card Types

| Type | Count Source | CTA |
|------|--------------|-----|
| Needs Review | Items with `status=pending` in review queue | "Go to Review" |
| Needs Approval | Agent runs awaiting human approval | "Go to Agents" |
| Policy Violations | Flagged items (PII, compliance) | "View Violations" |

### Card Layout

```
┌─────────────────────────┐
│ [icon] Needs Review     │
│         4 items         │
│   [→ Go to Review]      │
└─────────────────────────┘
```

### Empty State

When count is 0:
- Show warm message: "All caught up! No items need your attention."
- Still show the card for consistency

---

## Exit Criteria

- [ ] Shows count for each queue type
- [ ] Each card has icon, title, count, and CTA button
- [ ] Clicking CTA navigates to correct view
- [ ] Empty state shows warm message when count is 0
- [ ] Cards use `@open-hax/uxx` Card and Button primitives
- [ ] Responsive: cards stack on narrow viewports

---

## Out of Scope

- Review queue item list (see Review Queue view)
- Policy violation details
- Real-time count updates (polling ok)
