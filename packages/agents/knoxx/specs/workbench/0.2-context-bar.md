# Context Bar Navigation

**Parent**: `knowledge-ops-workbench-ux-breakdown.md`
**Epic**: 0. Shell Foundation
**Points**: 2
**Dependencies**: 0.1 Shell Layout Component

---

## Purpose

Left rail navigation with six named view entries and keyboard navigation.

---

## Scope

Create the navigation component that lets users switch between the six named views.

---

## Files to Create

| File | Purpose |
|------|---------|
| `src/shell/ContextBar.tsx` | Navigation component with view entries |
| `src/shell/nav-items.ts` | Navigation configuration (routes, labels, icons) |

---

## Behavior

### Navigation Items

| View | Route | Icon | Shortcut |
|------|-------|------|----------|
| Dashboard | `/` | `LayoutDashboard` | `SPC d` |
| Content Editor | `/content/:id?` | `FileText` | `SPC c` |
| Review Queue | `/review` | `CheckSquare` | `SPC r` |
| Memory Inspector | `/memory` | `Network` | `SPC m` |
| Agent Workspace | `/agents` | `Bot` | `SPC a` |
| Ops Log | `/ops` | `Activity` | `SPC o` |

### States

- **Active view**: Highlighted background, bold text
- **Hover**: Subtle background change
- **Collapsed mode**: Icons only with tooltips on hover

### Keyboard Navigation

- Arrow up/down: Move focus between items
- Enter: Activate focused item
- Escape: Clear focus

---

## Exit Criteria

- [ ] All six views reachable from Context Bar
- [ ] Active view highlighted with distinct styling
- [ ] Hover state shows subtle feedback
- [ ] Keyboard navigation (arrow keys) works
- [ ] Collapsed mode shows icons only
- [ ] Tooltips appear on hover in collapsed mode
- [ ] Click navigates to correct route

---

## Out of Scope

- Content of any view
- Status Bar
- Inspection Panel
- Route implementation

---

## Acceptance Tests

1. Click Dashboard icon → navigate to `/`
2. Click Memory Inspector icon → navigate to `/memory`
3. Press arrow down → focus moves to next item
4. Press Enter → navigate to focused item
5. Collapse sidebar → icons remain visible, labels hidden
6. Hover collapsed icon → tooltip shows view name
