## Behavior

### Suggestion Appearance

```
│  Body                                           │
│  ┌─────────────────────────────────────────────┐│
│  │ The Lake is the spine of the system.       ││
│  │                                             ││
│  │ [AI: Expand on data flow] [Accept][Revise] ││
│  │                              [Dismiss]      ││
│  │                                             ││
│  │ OpenPlanner handles event ingestion...     ││
│  └─────────────────────────────────────────────┘│
```

### Suggestion Types

| Type | Trigger | Action |
|------|---------|--------|
| Expand | AI detects thin section | Generate expansion |
| Clarify | AI detects ambiguity | Suggest clarification |
| Correct | AI detects error | Suggest correction |
| Complete | AI detects incomplete thought | Suggest completion |

### Interaction

- Hover: Shows diff preview of what would change
- Accept: Replaces text with suggestion, removes chip
- Revise: Opens inline prompt to modify suggestion
- Dismiss: Removes chip without changes

### Keyboard Chords

| Action | Chord |
|--------|-------|
| Accept suggestion | `SPC i a` |
| Revise suggestion | `SPC i r` |
| Dismiss suggestion | `SPC i d` |

## Exit Criteria

- [ ] Suggestions appear inline in body
- [ ] Diff preview shows on hover
- [ ] Accept button replaces text
- [ ] Revise button opens prompt
- [ ] Dismiss button removes chip
- [ ] Keyboard chords work
- [ ] Multiple suggestions can exist simultaneously
- [ ] Suggestion state persists across save

---

## Out of Scope

- Generating suggestions (backend concern)
- Suggestion quality/tuning
- Custom suggestion types

---

## Acceptance Tests

1. Load document with suggestions → chips appear inline
2. Hover chip → shows diff preview
3. Click Accept → text updates, chip disappears
4. Click Revise → prompt opens
5. Press `SPC i a` → accepts focused suggestion
6. Save document → suggestions preserved
