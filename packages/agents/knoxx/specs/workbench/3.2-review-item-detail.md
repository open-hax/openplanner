## Behavior

### Detail Layout

```
┌─────────────────────────────────────────────────────────────────┐
│ Item: synthesis-q4-summary.md                                   │
│ Type: agent synthesis    Confidence: 0.71                       │
├─────────────────────────────────────────────────────────────────┤
│ Output                                                          │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ # Q4 Summary                                                │ │
│ │                                                             │ │
│ │ The quarter saw significant growth in...                    │ │
│ │                                                             │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ Labels / Corrections                                            │
│ correctness:    [good ▾]                                        │
│ groundedness:   [grounded ▾]                                    │
│ notes:          [_________________________________]             │
│                                                                 │
│ [Approve]  [Needs Edit]  [Reject]  [Skip]                      │
└─────────────────────────────────────────────────────────────────┘
```

### Label Dimensions by Type

**Synthesis items:**
- correctness: good / partially-correct / incorrect
- groundedness: grounded / partially-grounded / hallucinated
- risk: safe / needs-review / dangerous

**MT segments:**
- adequacy: complete / partial / missing
- fluency: natural / acceptable / poor
- terminology: correct / inconsistent / wrong
- risk: safe / sensitive

**Ingestion items:**
- relevance: relevant / partially-relevant / irrelevant
- quality: good / acceptable / poor
- PII flag: clean / flagged

### Actions

| Action | Result |
|--------|--------|
| Approve | Item approved, moves to next |
| Needs Edit | Opens in scratchpad for rework |
| Reject | Item rejected, writes correction (3.3) |
| Skip | Moves to next item |

### Keyboard Chords

| Action | Chord |
|--------|-------|
| Approve | `SPC a` |
| Reject | `SPC r` |
| Skip | `SPC s` |

- [ ] Shows full output content

## Out of Scope

- Correction write-back (see 3.3)
- Multi-item selection
- Edit in place

## Acceptance Tests

1. Load synthesis item → shows correctness/groundedness/risk labels
2. Load MT item → shows adequacy/fluency/terminology labels
3. Click Approve → item approved, next item shown
4. Click Reject → item rejected, correction recorded
5. Press `SPC a` → approves current item
6. Enter notes → notes saved with review
