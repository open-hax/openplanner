# recovered cephalon-clj

This directory is an **archive**, not a runnable package.

It preserves the shape of the lost `cephalon-clj` branch closely enough that later work can recover topology, interfaces, and doctrine without pretending the original sources survived.

## Reading order

1. `docs/INDEX.md`
2. `docs/ARCHAEOLOGY.md`
3. `spec/architecture.md`
4. the recovered `*.md` source stubs under `cephalon-clj-brain/`, `cephalon-clj-discord-io/`, and `cephalon-clj-shared/`

## What this archive remembers

- the two-process split between brain and Discord IO
- a shared wire / RPC layer
- MCP and toolset topology
- prompt-path and agent-registration clues for the Duck branch

## What it does not claim

- that the original code was fully recovered
- that these markdown stubs are runnable source
- that the archive should outrank the living TS/CLJS/CLJ packages
